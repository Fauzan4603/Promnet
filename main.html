Git adalah sistem kontrol versi yang sangat populer yang digunakan oleh pengembang perangkat lunak untuk mengelola perubahan dalam kode sumber proyek mereka.
1. Pengenalan Git:
Git adalah sistem kontrol versi yang diciptakan oleh Linus Torvalds pada tahun 2005. Ini digunakan untuk melacak perubahan dalam kode sumber proyek perangkat lunak.

2. Repositori Git:
Repositori Git adalah tempat penyimpanan untuk kode sumber proyek. Ada dua jenis repositori: lokal (di komputer pengembang) dan repositori jarak jauh (biasanya di platform seperti GitHub atau GitLab).

3. Commit:
Commit adalah tindakan menyimpan perubahan dalam repositori Git. Setiap kommit memiliki pesan yang menjelaskan perubahan yang dilakukan.

4. Branch (Cabang):
Cabang adalah salinan independen dari repositori utama yang memungkinkan pengembang untuk bekerja pada fitur atau perbaikan bug tanpa mengganggu kode sumber utama. Perubahan pada cabang ini dapat diintegrasikan kembali ke cabang utama nanti.

5.Perintah Utama Git
- `git init`: Inisialisasi repositori Git lokal.
- `git clone`: Menduplikasi repositori Git yang ada.
- `git add`: Menambahkan perubahan ke dalam staging area.
- `git commit`: Menyimpan perubahan yang ada di staging area.
- `git push`: Mengunggah perubahan ke repositori jarak jauh.
- `git pull`: Mengambil perubahan dari repositori jarak jauh.
- `git branch`: Menampilkan daftar cabang.
- `git merge`: Menggabungkan perubahan dari satu cabang ke cabang lain.
- `git checkout`: Berpindah antara cabang atau commit.
- `git status`: Menampilkan status perubahan dalam repositori.

6. Konflik Gabungan (Merge Conflict):
Terjadi ketika Git tidak dapat secara otomatis menggabungkan perubahan dari dua cabang yang berbeda, dan memerlukan intervensi manual dari pengembang untuk menyelesaikannya.

7. GitHub dan GitLab:
Platform hosting seperti GitHub dan GitLab memungkinkan kolaborasi tim, pelacakan isu, pembuatan permintaan tarik (pull request), dan banyak lagi dengan menggunakan Git sebagai dasar.
